# AGENTS.md — arikoi 開発原則

arikoiは**ティラノスクリプト作品リポジトリ**である。ゲームエンジンではない。
実装すべきものはシナリオ・キャラクター・分岐・演出・UI・アセットであり、
ノベルゲームエンジンそのものを実装することではない。

## 原則

1. **ティラノ標準機能を最優先する。** セーブ/ロード・バックログ・オート・スキップ・
   メッセージウィンドウ・選択肢・レイヤー管理・シーン遷移・サウンド管理・背景/キャラ表示は
   ティラノスクリプトの標準機能を使う。これらを独自実装しない
2. **独自実装より既存機能を優先する。** 新しい実装を追加する前に必ず確認する:
   (1) ティラノ標準機能で実現できるか (2) 既存プラグインで実現できるか (3) 作品固有の処理か。
   3つすべてを満たした場合のみ独自実装を追加する
3. **汎用化しない。** 他作品で使うことを前提とした抽象化・ライブラリ化はしない。
   この作品に最適な実装を優先する
4. **中間DSLを作らない。** シナリオのソースはティラノスクリプトそのもの。
   Markdown正本・独自タグ(`#scene` `#character` `#choice` のような)・AI再入力専用形式を導入しない
5. **`.ks` を正本とする。** `data/scenario/*.ks` は生成物ではなく正本であり、人間が自由に編集する。
   scenario-to-tyrano で生成した場合も、生成後の `.ks` が正となる
6. **必要になるまで拡張しない。** 「将来使うかも」で機能・構造・設定を先回りして作らない

## リポジトリ固有の注意

- `tyrano/`(エンジン本体)は原則無改造。例外は次の2種のみ:
  (1) `tyrano/lang.js`(ダイアログ文言の英語化)
  (2) `[arikoi security patch]` コメントを付けたセキュリティパッチ
  (kag.tag_ar.jsのQR URLスキーム検証、kag.tag_system.jsの入力値JSONリテラル化)。
  エンジンをバージョンアップする際は `git grep "arikoi security patch"` とlang.jsの差分を再適用すること
- ティラノのタグ属性値内の半角スペースはパーサに除去される。`&nbsp;` で表現する
  (例: `[glink text="Ask&nbsp;her&nbsp;what&nbsp;it&nbsp;is"]`)
- ゲーム内テキスト・UI文言は英語。開発会話・コミットメッセージ・docsは日本語
- 変更後は `npm run e2e` のグリーンと、`npm run serve` でのブラウザ実機確認を行う
  (型チェックが存在しない構成のため、実機確認の比重が高い)
- 文字送り中(`is_adding_text=true`)にセーブするとロード後にクリック受付が壊れる
  (エンジン仕様)。E2Eではセーブ前に `waitForTextSettled` を挟む
