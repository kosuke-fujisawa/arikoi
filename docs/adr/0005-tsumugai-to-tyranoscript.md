# 0005. tsumugai + 独自Webランタイムを廃止し、ティラノスクリプトを実行基盤とする

- 日付: 2026-07-19
- 状態: 採用

## 背景

ADR 0001の方針転換後、arikoiは「tsumugai(Rust製CLI)でシナリオDSLをStoryBundle JSONへコンパイルし、Svelte + Vite + TypeScript製の独自Webランタイムで再生する」構成で技術スパイクを完了していた(#24〜#34、#43、#58)。ゴールデンパス(start→advance→choose→save→reload→load→continue→ending)のE2Eも通っていた。

一方で、この構成には本質的な問題があった。

- **作品ではなくエンジンを作っていた**: セーブ/ロード・バックログ・テキスト送り・選択肢UI・構造検証などの実装・テスト(vitest 41件 + E2E)は、いずれも既存のノベルゲームエンジンが標準機能として持つものの再発明だった
- 独自DSL(tsumugai記法)→ StoryBundle JSON → 独自ランタイムという多段の変換契約(ADR 0002〜0004)の維持コストが、シナリオ・キャラクター・演出という作品本体の制作を圧迫していた
- tsumugaiのバージョン固定・サブプロセス連携(#57、tsumugai#128)など、リポジトリ間の統合コストが常に発生していた

制作の目的は「ノベルゲームエンジンを作ること」ではなく「ありふれた恋のカタチ(A Familiar Shape of Love)」という作品を完成させることである。

## 選択肢

1. **現行構成(tsumugai + 独自ランタイム)を継続する**
   スパイクは成功していたが、本編制作に進むほど標準VN機能の再発明(オート・スキップ・既読管理・コンフィグ等)が積み上がる。
2. **Ren'Pyへ回帰する**
   ADR 0001で棄却済み。ブラウザ配布(itch.io)との相性が悪い。
3. **ティラノスクリプトを実行基盤とし、arikoiをティラノ作品リポジトリにする(採用案)**
   セーブ/ロード・バックログ・オート/スキップ・選択肢・メッセージウィンドウ等が標準搭載で、ブラウザ配布が正規の配布形態。日本製ギャルゲー文法との親和性も高い。

## 採用案

- 実行基盤はティラノスクリプト(ShikemokuMK/tyranoscript)v514。エンジン一式(`index.html` + `tyrano/` + `data/`)をリポジトリルートにコミットして同梱する(公式ライセンスは作品配布目的の同梱を許容。`LICENCE.txt`・`readme.txt`を保持)
- シナリオの正本は `data/scenario/*.ks`(ティラノスクリプト記法)。中間DSL・Markdown正本・AI再入力専用形式は作らない。`.ks` は生成後も人間が直接編集する
- シナリオの新規執筆は「自然言語の原稿 → [scenario-to-tyrano](https://github.com/kosuke-fujisawa/scenario-to-tyrano) → `.ks`」の流れを想定する(同ツールは現状スタブのため、それまでは手作業で書く)
- エンジン(`tyrano/`)は原則無改造。唯一の例外は `tyrano/lang.js` のダイアログ文言英語化(エンジン更新時は差分を再適用する)
- 独自実装を追加できるのは「ティラノ標準機能で実現できない」「既存プラグインでも実現できない」「作品固有の処理である」の3条件を満たす場合のみ
- テストはPlaywright E2E(ゴールデンパス+セーブ/ロード)のみ維持する。エンジン内部のユニットテストは持たない(エンジンはティラノに委譲したため対象が存在しない)

## 採用理由

- 標準VN機能の再発明を全廃し、実装対象をシナリオ・キャラクター・分岐・演出・UI・アセットという作品本体に限定できる(Product First)
- ビルド工程が消滅する(コンパイル・バンドル不要。リポジトリをHTTP配信すればそのまま動き、zip化すればそのままitch.io配布物になる)
- ティラノは商用作品での実績が長く、セーブ互換・ブラウザ互換の枯れた実装を持つ

## 欠点・リスク

- 独自ランタイム(src/、tests/、ADR 0002〜0004の設計)は破棄する。ただしゴールデンパスのE2E設計・シナリオ本文・キャラクター設定はティラノ構成へ引き継いだ
- エンジンのバージョン更新はzip差し替え+lang.js差分再適用の手作業になる
- タグ属性値内の半角スペースがパーサに除去される等、ティラノ固有の癖への対応が必要(`&nbsp;`で表現)
- v5.13bに任意コード実行の脆弱性報告があった(v514を採用。エンジン更新時はセキュリティ情報を確認する)

## 影響範囲

- 削除: `src/`(engine層+Svelte UI)、`tests/`(vitest)、`scripts/compile-story.ts`、`scripts/install-tsumugai.sh`、`tools/`、`public/`、vite/svelte/tsconfig設定群、`games/familiar-shape-of-love/`(内容は移植済み)、ルート`scenarios/`・`assets/`
- ADR 0002〜0004は失効(前提とする独自ランタイム/StoryBundleが廃止)。ADR 0001の「ブラウザVN・英語圏itch.io」の方向性は有効のまま、実装手段のみ本ADRで変更
- CI: scenario-check.yml / web-ci.yml → game-ci.yml(E2E + 配布zipアーティファクト)
- epic #23の「独自runtime」前提の記述は本ADRで訂正。#19(素材)の配置先は `data/bgimage/` 等のティラノ標準フォルダに変更
- 関連リポジトリ: kosuke-fujisawa/tsumugai はarikoiからの依存が消滅(tsumugai自体の存続はtsumugai側の判断)
