# 要求仕様書 v0.1 実装状況整理

本書は、ユーザーから提示された「ありふれた恋のカタチ」要求仕様書 v0.1(プロジェクト概要編・システム仕様編)の各項目を、現状のリポジトリ実装・ADR・既存ドキュメント資産と突き合わせて整理したものである。

要求仕様書システム仕様編 32章「実装開始前の調査事項」が求める「踏襲対象リポジトリの調査 → 本書との差分整理 → 既存実装を優先する」という手順の、現時点での回答に相当する。

**前提となる方針転換(要求仕様書執筆後に確定した決定)**

- 実行基盤は独自Webランタイム(Svelte+tsumugai)ではなく **ティラノスクリプト v514** に全面移行済み([ADR 0005](../adr/0005-tsumugai-to-tyranoscript.md))。ゲーム内テキスト・ターゲットは英語圏itch.io向けに転換済み([ADR 0001](../adr/0001-renpy-to-browser-vn-pivot.md))。
- 開発原則の正本は [AGENTS.md](../../AGENTS.md): 「ティラノ標準機能を最優先し、独自実装しない」「汎用化しない」「中間DSLを作らない」。要求仕様書が想定する「共通基盤の踏襲」は、実質的に**ティラノスクリプト標準機能をそのまま使う**ことで置き換えられている。
- 姉妹作品(negaboku, chiriyuku-monotachi)とも**ランタイムは共有せず、開発運用知見のみ共有**する方針([ADR 0006](../adr/0006-share-tyrano-development-practices-not-runtime.md))。要求仕様書23章が想定する「共通テンプレートリポジトリ/Git Submodule」方式は採用されていない。
- 舞台は現代日本の高校から**魔法大学**へ変更し、キャラクターは**6人の攻略ヒロインと非攻略の友人ミユキ**へ再構成した([ADR 0007](../adr/0007-magic-university-seven-character-cast.md))。現行Kaedeプロローグは技術検証としては動作するが、作品版の本編としては差し替え対象である。

**状態タグの凡例**

| タグ | 意味 |
|---|---|
| 実装済み(標準) | ティラノスクリプト標準機能でそのまま満たされている |
| 実装済み(未接続) | ロジック/APIは存在するが、本編シナリオ(title.ks等)から呼ばれておらず到達不能 |
| 一部実装 | 部分的に存在するが要求仕様の水準に未達 |
| 未実装 | 該当する実装が存在しない |
| 方針転換 | 要求仕様書の想定と異なる方針が既に確定している |
| 参考資料として現存 | 転換前(旧Ren'Py構想)の資料としてdocs/配下に残るが実装対象は未確定 |
| 要確認 | 調査では事実確認できず、ユーザー確認が必要 |

## 既知の不整合・技術的負債

今回の調査で判明した、要求仕様書とは別軸のドキュメント/実装間の不整合。今後の作業時に注意が必要。

1. **README.md記載とdata/実態の乖離**: README.mdのディレクトリ構成説明には`fgimage/`, `bgm/`, `sound/`が挙げられているが、実際は`fgimage/`・`bgm/`は空、`sound/`は**ディレクトリ自体が存在しない**。
2. **`scripts/lightweight-review.sh`とpackage.jsonの不整合**: 同スクリプト(および連動する`.github/workflows/lightweight-review.yml`、現状ともにuntracked)は`npm run check` / `npm test` / `npm run build` / `npm run textlint`を呼び出す設計だが、現行`package.json`にはこれらのスクリプトが**一つも定義されていない**(`serve`, `e2e`, `release:zip`のみ)。ADR 0005のティラノ移行(Svelte/vitest/vite一式削除)以前の名残と推測され、現状のままCIに乗せると失敗する可能性が高い。
3. **AGENTS.mdの未コミット差分**: 作業ツリー上でAGENTS.mdに変更があり(ADR 0006への参照追加)、対応するADR 0006ファイル自体もuntrackedの状態。
4. **`.review/`配下の古い実行ログ**: `type-check.log`, `unit-tests.log`, `production-build.log`, `scenario-textlint.log`等がSvelte/vitest時代のものとしてリポジトリに残存しており、現行構成とは対応しない。
